# JavaBean

Defínese un JavaBean (ou Bean) como un compoñente software reutilizable e que pode ser manipulado visualmente por unha ferramenta de programación en linguaxe Java. Para que unha clase sexa considerada un JavaBean debe cumprir unha serie de normas. Estas normas permitirán ao IDE interrogar o Bean e coñecer as propiedades que define e os tipos de eventos que pode xerar en resposta a diversas accións.

# Persistencia

A persistencia é a característica dunha clase que permite almacenar o estado para que perdure a través do tempo. Para implementala, é necesario que poida ser almacenada nun arquivo e recuperada posteriormente. O mecanismo que implementa a persistencia chámase serialización. E ao proceso de almacenar o estado dunha clase nun arquivo chámase serializar. Todos os compoñentes deben persistir. Para iso, dende o punto de vista de Java, deben implementar a interface java.io.Serializable que ofrece a posibilidade de serialización automática.

# Introspección

Permitir ao IDE poder recoñecer elementos de deseño como os nomes das propiedades, xunto co tipo de datos e os métodos.

# Personalización

A aparencia e conduta do Bean pode ser alterada pola persoa programadora mediante os cambios dos valores nas propiedades.

# Eventos

Debe informar ao IDE dos sucesos que pode xerar en resposta ás accións das persoas usuarias e tamén os eventos que pode manexar.
